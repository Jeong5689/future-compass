주요 작업 이력
1) 의존성 설치 및 타입 검사
  - 명령: `npm install`
  - 명령: `npm run lint` (`tsc --noEmit`) -> 타입 검사 통과

2) 런타임 개발 서버
  - 명령: `npm run dev` -> 개발 서버 실행 확인 (http://0.0.0.0:3000)

3) 코드 수정을 시도함
  - 파일: `src/App.tsx`
  - 변경: `ChevronRight` 중복 alias 제거 및 사용 통일 (일부 수정이 유저에 의해 되돌려진 기록 있음)

4) 프로덕션 빌드 및 서버 번들 생성
  - 명령: `npm run build` (Vite 빌드 + esbuild 서버 번들)
  - 결과: `dist/` 산출물 생성, `dist/server.cjs` 생성 완료

5) 번들된 서버 실행(통합 검증)
  - 명령: `npm run start`
  - 결과: 서버 실행 확인 → 출력: "[Future Compass] Server running on http://0.0.0.0:3000"

6) API 및 브라우저 점검
  - 점검: `curl -I http://127.0.0.1:3000/` -> 200 OK
  - 점검: `curl -I http://127.0.0.1:3000/index.html` -> 200 OK
  - 점검: `/api/gemini/chatbot` -> 데모 응답 JSON (GEMINI_API_KEY 미설정 시 시뮬레이션)
  - 점검: `/api/gemini/career-analysis` -> 데모 JSON (요청 바디 전달 후 정상 JSON 반환)
  - 점검: `/api/gemini/futuristic-career` -> 데모 JSON
  - 메모: Windows CMD에서 `curl` 사용 시 JSON 인용 문제로 `Invalid messages array.` 같은 오류 발생함. PowerShell 권장 혹은 큰따옴표 이스케이프 필요.
